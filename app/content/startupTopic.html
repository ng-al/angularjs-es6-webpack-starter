Start Up

When a webpack application starts, the initial template and corresponding code file are loaded.

The template /app/index.html is very minimal. None of the usual style or script loading is present; instead, those items are loaded vie require statements within the code. Also note, that the ng-app directive is missing, AngularJS bootstrapping is handled in code. The only AngularJS directives in index.html is left-nav and ng-view.

The startup code in /app/index.es6 is responsible for bootstrapping the entire application:

  1. The bootstrap and font-awesome CSS are loaded.
  2. AngularJS is loaded.
  3. /app/main/main.es6 is imported, this initializes the module. See Main.
  4. Lastly, AngularJS is attached to the <body> tag.
