LeftNav

/app/leftNav/leftNav.feature.es6 is the "main" file for the left navigation feature. By importing this file, the feature will be set up. This file performs the following tasks:

  1. Imports the stylesheet for the feature.
  2. Imports the leftNav module. (This creates it.)
  3. Imports the leftNav component (directive).
  4. Registers the leftNav component with the leftNav module.

/app/leftNav/leftNav.component.es6 is the leftNav Angular directive. It is written in component-style, which simply means that the directive uses a separate template and controller. The component's file has a controller class defined at the top, and the directive class at the bottom. (The template is in a separate file /app/leftNav/leftNav.html and is included by the require statement on line 46.

The LeftNav Directive

Note that the "normal" directive API properties are defined as properties of the class instance. The link function (not used in this example), is similarly defined. The controller is associated with the directive on line 41 and 42. If you examine the template, you will see that all of the view model properties (defined by the controller), are referenced in the template with the prefix vm.

this.scope = false
Since all of the work is performed in the controller, there is no need to create an isolated scope for the directive. Note that when you do not create a scope, the scope parameter in the link function is the parent element's scope.

this.bindToController = {}
This creates an isolated scope for the controller. Note that any properties created in this object will be present in the controller as properties on the $scope object.

The LeftNav Controller

The routing service is used to get the array of routing items. This collection is added to the view model so that the template can perform an ng-repeat over the data.

A watch is placed on $location.path() so that whenever the path changes, the activeTabIndex will be set.

onTabClick() is called whenever a user clicks one of the "pills."
